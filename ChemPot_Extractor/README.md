
# 🧪 VASP Chemical Potential & Inequality Extractor

### 1. Introduction
This Python tool automates **Chemical Potential** analysis for **Defect Formation Energy** and **Phase Stability** studies. It batch-processes VASP `scf` results to extract atomic compositions and total energies, generating linear inequalities and structured Excel reports.

**Core Functions:**
* **Auto-Parsing**: Reads stoichiometry from the 1st line of `POSCAR`.
* **Energy Extraction**: Captures `energy without entropy` from `OUTCAR`.
* **Inequality Construction**: Generates stability constraints (e.g., $1 \cdot \mu_{Pb} + 2 \cdot \mu_{I} \le E_{tot}$).
* **Dual Output**: Exports results to both `.xlsx` (spreadsheet) and `.txt` (raw text) formats.

## 1. 脚本简介

本 Python 脚本专为 **缺陷形成能（Defect Formation Energy）** 或 **表面相图** 计算中的“化学势（Chemical Potential）”分析而设计。
它可以批量遍历多个相（如单质、二元/三元化合物）的 VASP `scf`（自洽场计算）结果，自动提取各相的原子组成与总能量，并一键生成用于求解相图边界的**线性不等式组**和**数据汇总 Excel 表格**。

**核心功能：**
1. **自动读取结构**：从 `POSCAR` 第 1 行自动解析包含的目标元素及其原子个数。
2. **自动提取能量**：从 `OUTCAR` 中精准抓取最后一步的 `energy without entropy`。
3. **自动构建不等式**：将原子数和能量拼接成热力学稳定条件不等式（例如：`2*x_Pb + 4*x_I <= -20.534`）。
4. **双格式输出**：同时生成结构化数据的 `.xlsx` 表格和纯文本的 `.txt` 不等式组文件。

## 3. 严格的目录与文件规范 (⚠️ 极重要)

脚本的正常运行极度依赖于您的文件目录结构和 `POSCAR` 格式，请务必遵守以下规范：

### 📁 3.1 目录拓扑结构
脚本会在您指定的 `base_path` 下，去寻找各个物质文件夹内部的 `scf` 子文件夹。
标准的目录结构必须如下所示：
```text
chem_pot/                <-- (base_path 指向这里)
├── C/
│   └── scf/
│       ├── POSCAR       <-- (必须存在)
│       └── OUTCAR       <-- (必须存在)
├── H2/
│   └── scf/
│       ├── POSCAR
│       └── OUTCAR
└── PbI2/
    └── scf/
        ├── POSCAR
        └── OUTCAR
```

### 📄 3.2 POSCAR 格式要求（核心逻辑）
脚本是通过正则匹配提取原子个数的，因此**每一个 `POSCAR` 的第一行必须清楚地写明化学式及其数量**。
* **正确的第一行格式示例**：`C 1`、`H 2`、`Pb1 I2`、`H 3 C 3 N 5`
* **错误的第一行格式示例**：`System`、`fcc Si`、`Generated by VESTA`（如果第一行是这些，脚本将抓取不到原子数量，对应不等式会生成失败）。

## 4. 脚本参数修改指南

在运行脚本前，请使用文本编辑器打开 Python 脚本，并根据您当前的计算任务修改**「配置区域」**的三个核心参数：

### 🔧 1. `base_path` (主目录路径)
* **作用**：指定存放所有化合物文件夹的总目录。
* **修改示例 (Windows)**：`base_path = r"C:\Users\123\Desktop\Project\My\Workspace\Gua\Gua_dfe\chem_pot"` *(注意前面的 `r` 不能删，用于防止路径转义)*
* **修改示例 (Linux/Mac)**：`base_path = "/home/user/Project/chem_pot"`

### 🔧 2. `folder_list` (待提取的文件夹列表)
* **作用**：告诉脚本需要遍历哪些物质的文件夹。
* **修改建议**：确保列表中填写的名字与真实的文件夹名字**完全一致**（区分大小写）。如果增加或删除了某个相的计算，请同步修改此列表。

### 🔧 3. `target_elements` (目标元素种类)
* **作用**：决定 Excel 表格的列名，以及不等式中的未知数 $x_i$ 的名称。
* **修改示例**：`target_elements =["C", "H", "N", "Pb", "I"]`。
* **注意**：脚本只会提取这里列出的元素。如果 `POSCAR` 中含有 `O` 元素，但您没有将其写进 `target_elements`，那么 `O` 将会被忽略。

## 5. 运行脚本与产出文件说明

配置好上述参数后，直接运行脚本：

终端会实时打印提取进度和状态（`OK` 或 `Missing`）。运行结束后，会在脚本所在目录下生成两个文件：

直接生成可用于数学软件（如 MATLAB, Mathematica）或 Python 凸包计算代码的不等式组。
*内容示例：*
```text

2*x_H <= -6.78901230
1*x_Pb + 2*x_I <= -15.43210980
```

## 6. 常见排错指南 (Troubleshooting)

1. **表格中元素数量全为 0，Equation 显示 "No Atoms Found"**
   * **原因**：`POSCAR` 的第一行没有写明元素和数字。
   * **解决**：手动把每个 `POSCAR` 的第一行改成 `元素符号+数字`（例如 `Pb 1 I 2`）。
2. **终端状态显示 "Missing"，Energy 为空**
   * **原因**：`OUTCAR` 文件不存在，或者 VASP 计算中途崩溃，尚未输出 `energy without entropy` 这一行。
   * **解决**：检查对应相的计算是否完成，或确认目录结构是否严格满足 `文件夹名/scf/OUTCAR`。
3. **报错 `ModuleNotFoundError: No module named 'pandas'`**
   * **原因**：Python 环境缺失必须的第三方库。
   * **解决**：运行 `pip install pandas openpyxl` 进行安装。
